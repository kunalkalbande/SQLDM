Set your default alert configuration

SQLdm allows you to customize the thresholds that generate informational, warning, and critical alerts and notifications. In addition, you can add comments to the alert that appear whenever an alert is generated. SQLdm provides you with the ability to create a default alert configuration so that all new SQL Server instances you add to SQLdm use the same template, reducing the amount of time it takes you to tweak alert configurations.

TIP

Make sure you review the list of every default alert ID, its description, and what events are associated with it before changing your alert configuration.

Customize your alerts

The Alert Configuration window allows you to customize the thresholds that generate informational, warning, and critical alerts and notifications for all of your currently monitored metrics.

To customize your alert configuration:

  1. Right-click the SQL Server instance you want to edit from the Servers tree, and then select Configure Alerts.
  2. Select each of the metrics you want to edit from the list. If you are satisfied with how your metrics are set up, skip steps 4 and 5.
  3. Check the Informational, Warning, and Critical check boxes to include alerts for these states.
  4. Change the alert thresholds by moving the arrows to the appropriate levels.
  5. Select the Comments tab, and then type the information you want displayed in the alert message for this metric.
  6. Click Apply.
  7. If you want to replicate your edits to other monitored SQL Server instances, tags, or templates, click Yes.
  8. Click OK to accept your changes.

TIP

Note that when you modify an alertable metric in the Alert Configuration window, the Create Template button is disabled. Once you apply the modification, this command button is enabled again.

Set up your default alert configuration

The Default Alert Configuration window allows you to configure standard alerts for every SQL Server instance that is added to SQLdm. SQLdm uses this alert template each time you register a new SQL Server instance so that you can use all the same alert thresholds on the collected metrics.

To set up your alert configuration:

  1. Select a SQL Server from the Servers tree to use as the baseline for your template.
  2. Select Tools > Default Alert Configuration.
  3. Select each of the metrics you want to edit from the list. If you are satisfied with how your metrics are set up, skip steps 4 and 5.
  4. Change the alert thresholds by moving the arrows to the appropriate levels.
  5. Select the Comments tab, and then type the information you want displayed in the alert message for this metric.
  6. Click Apply.
  7. If you want to replicate your edits to other monitored SQL Server instances, tags, or templates, click Yes.
  8. Click OK to accept your changes.
