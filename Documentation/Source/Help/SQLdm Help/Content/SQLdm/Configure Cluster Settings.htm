Configure cluster settings

If you have a clustered environment, SQLdm can trigger an alert whenever a cluster is running on a non-preferred node. The preferred cluster node is the "active" node in a cluster. By default, the preferred cluster node is the first node added when deploying SQLdm in your clustered environment.

Access the Cluster Settings tab

You can access the Cluster Settings tab of the Monitor SQL Server Properties window by right-clicking the appropriate monitored SQL Server instance, and then selecting Properties. Click Cluster Settings when SQLdm displays the Monitored SQL Server Properties window.
