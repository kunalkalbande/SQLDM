Customize disk statistics

Disk drive statistics collection provides the ability for SQLdm to alert on disk drives and database space calculations. By default, SQLdm attempts to auto-discover local disk drives on monitored instances. In some environments, the auto-discovered list may be incomplete or incorrect. The Customize disk drive statistics collection window allows you to turn off the auto-discover feature and manually add the drives for drive statistics collection.

TIP

SQLdm populates the disk drive list using operating system metrics from a WMI call even if your collection is configured to use OLE automation. SQLdm requires you to enter credentials to connect directly to WMI. This functionality lets SQLdm quickly discover mount points while the OLE automation does not provide such information.

Access the Disk Drives tab

You can access the Disk Drives tab of the Monitor SQL Server Properties window by right-clicking the appropriate monitored SQL Server instance, and then selecting Properties. Click Disk Drives when SQLdm displays the Monitored SQL Server Properties window.

Incomplete disk drive list

SQLdm automatically discovers local disk drives on your monitored instances. Sometimes this list is incomplete or contain drives that are not related to SQL Server. If you need to modify this list, clear the Discover disk drives check box, and then type the drive information including mount points, if necessary, into the Available disk drives field. Select the appropriate drive or drives and then click Add to move the drive information to the Selected disk drives box. Click OK.
