Top Databases

The Top Databases report identifies your worst performing SQL Server databases based on the size of the database, the growth rate of the database, and the number of reads, writes, and transactions per second performed on the database.

You can further filter your monitored databases by wait time with waits that exceed a defined Wait Threshold in milliseconds.

Use this report to compile a list of databases that used most often or have the heaviest loads.

TIP

You can use % as a wildcard in the Search Value field.

When to run this report

You should run the Top Databases report on a routine basis, such as once a week. This report helps you:

  • Plan for upcoming maintenance
  • Identify issues that may impact third-party application performance even though the host SQL Server instance remains healthy
  • Compare database performance across your enterprise

How SQLdm calculates metric values on this report

Most values on the Top Databases report correlate with the related SQL Server metric. However, some metric values represent averages over time. Likewise, a few metric values are based on performance algorithms that use multiple statistics to calculate a more accurate metric.

The following table provides additional information as to how SQLdm calculates that values in each column of the Top Databases report.

Column Name How SQLdm calculates this value

% Growth

Rate of growth of this database.

Bytes Read/sec

Amount of data read from this database, in bytes per second.

Bytes Written/sec

Amount of data written to this database, in bytes per second.

Database Name

Name of the database.

Data File Size

Size of the data file on this database.

Data Size

Amount of data stored in the data file.

Log File Size

Size of the log file on this database.

Transactions/sec

Number of transactions occurring on this database per second.
