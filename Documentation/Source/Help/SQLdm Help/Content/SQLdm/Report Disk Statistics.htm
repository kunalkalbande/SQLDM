Disk Statistics

The Disk Statistics report lets you view trends for the disk activity of a monitored SQL Server instance over a specified period of time. To successfully monitor OS metrics, make sure that you enable WMI or OLE automation on the monitored instance. Check Show Tabular Data to display results in a single tabular or grid format.

Check Show Baseline to include baseline data points in the Disk Statistics report. For additional information about baseline configuration, see Configure server baseline options.

When to run this report

You should run the Disk Statistics report to analyze the disk activity such as work files per minute or page read/writes per second.

How SQLdm calculates metric values on this report

Most values on the Disk Statistics report correlate with the related SQL Server metric. However, some metric values represent averages over time. Likewise, a few metric values are based on performance algorithms that use multiple statistics to calculate a more accurate metric.

The following table provides additional information as to how SQLdm calculates that values in each column of the Disk Statistics report.

Column Name How SQLdm calculates this value

% Disk Busy

Calculated as 100 – PercentIdleTime counter from Win32_PerfRawData_PerfDisk_PhysicalDisk, instance “_Total” – weighted average. Note that this counter is often inaccurate on Windows Vista and Windows 2008 and 2008 R2 operating systems. In this situation, SQLdm uses the value from the Win32_PerfFormattedData_PerfDisk_PhysicalDisk object.

Date

Timestamp when the Collection service gathered this data.

Disk Queue Length

From AvgDiskQueueLength counter from Win32_PerfRawData_PerfDisk_PhysicalDisk, instance “_Total” – weighted average.

IO Activity %

SQL Server IO busy percentage, derived from @@IO_BUSY in the same manner as sp_monitor.

Page Lookups Per Minute

From “page lookups/sec” counter from SQLServer:Buffer Manager object. Weighted average converted to per minute value.

Page Reads Per Second

From “page reads/sec” counter from SQLServer:Buffer Manager object. Weighted average.

Page Splits Per Minute

From “page reads/sec” counter from SQLServer:Access Methods object. Weighted average. This counter does not differentiate between “good” and “bad” page splits.

Page Writes Per Second

From “page writes/sec” counter from SQLServer:Buffer Manager object. Weighted average.

Temp DB Size (MB)

Collected database size of tempdb as a weighted average over the timeframe.

Work Files Per Minute

From “workfiles created/sec” counter from SQLServer:Access Methods object. Weighted average converted to per minute value.
